# DeltaCoder v1.3 — Phase 1: Config Update + Re-Pretokenize

**Status:** ⏳ PENDING — waiting for v1.2 SFT training to complete

**Goal:** Create `v1.3/scripts/pretokenize.py` (SEQUENCE_LEN=32768) and `v1.3/configs/axolotl.yaml`, then re-pretokenize the existing 230K dataset on the H200 instance.

**Why v1.3 exists:** v1.2 trained with `sequence_len: 8192`. The OCR dataset (nvidia/OpenCodeReasoning) averages 8,477 tokens per sequence with 50% of sequences exceeding 8,192 tokens. NVIDIA trained this dataset at 32K context. Training at 8192 truncated ~32,500 reasoning traces mid-thought — the model never saw the final answer for those examples. The H200 has 141GB VRAM but v1.2 only used 45GB. `seq_len: 32768` fits within VRAM, captures full OCR reasoning traces, and produces roughly 4x fewer but more token-dense steps — similar total training time, significantly better quality.

**Prerequisites:**
- v1.2 SFT training fully complete on H200
- v1.2 post-training steps done (merge, uploads, cache cleanup)
- H200 instance still alive at `root@213.5.130.43 -p 28988`
- `data/v1.2_sft_train.jsonl` already exists at `/workspace/DeltaCoder/data/v1.2_sft_train.jsonl` on the remote — **do not re-download or re-preprocess**
- `$HF_TOKEN` already set in `~/.bashrc` on remote (read/write token)

**The dataset does not change.** Same 230K rows, same 5 datasets as v1.2. Only the tokenization and training config change.

| `sequence_len` | 8192 | **32768** | Full OCR reasoning traces; H200 has headroom |
| `SEQUENCE_LEN` in pretokenize.py | 8192 | **32768** | Must match training config |
| Estimated steps | ~23,056 | **~5,800** | 4x larger windows = 4x fewer steps |
