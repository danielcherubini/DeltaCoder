# DeltaCoder v1.2 — Dataset Pruning Plan

> **Note:** This plan uses pre-restructure paths (`v1.1/`, `v1.2/`, `v1.3/`).
> The repo was restructured on 2026-04-06: `v1.1/` → `qwen3.5/v1.0/`, `v1.2/` → `qwen3.5/v1.1/`, `v1.3/` → `qwen3.6/v1.0/`.

**Goal:** Reduce training set from 262K to ~125K high-density rows while maintaining coverage of all three priorities: (1) Coding, (2) Tool Calling, (3) Agentic Workflows.

**Why:** Full 262K dataset takes ~176 hours on 1x H100 ($378). A 125K dataset should take ~67-80 hours (~$145-172), while potentially improving quality by removing redundancy and "syntax noise" that dilutes the model's existing knowledge.

**Principle:** Density over volume. Qwen3.5-9B already knows Python, JavaScript, etc. We don't need to teach it coding from scratch — we need to sharpen its tool calling, agentic behavior, and long-form reasoning.

## Filtering Criteria

### nemotron_tool_calling: 80K → 40K
**Strategy:** Filter for complexity — keep rows with more turns and more tool calls.
- Sort by conversation length (number of messages)
- Keep top 50% longest conversations
- This naturally selects multi-step tool-calling workflows over trivial single-call examples

### opencoder_reasoning: 50K → 25K
**Strategy:** Filter for reasoning depth — keep longest thinking traces.
- Tokenize assistant responses, measure length
- Keep only rows where assistant response is >2000 tokens (has meaningful reasoning)
- This ensures every example has a full begin→middle→end reasoning arc
- Directly addresses the v1.1 looping issue (model learned to start thinking but not to conclude)

### code_feedback: 40K → 15K
**Strategy:** Keep multi-turn iterations only.
- Filter for conversations with ≥4 messages (at least 2 rounds of user feedback)
- These teach the model to respond to critique and fix its own code
- Single-round "here's my review" examples are less valuable

### swesmith: 30K → 20K
**Strategy:** Keep longer trajectories with more tool calls.
- Sort by number of tool calls in conversation
- Keep top 67% (≥3 tool calls per conversation)
- Longer agentic trajectories teach the model to maintain context across many steps

### magicoder: 30K → 5K
**Strategy:** Aggressive quality filter — only the hardest problems.
- Tokenize the full conversation
- Keep only the top 5K longest (hardest problems with most detailed solutions)
- Qwen3.5 already has strong code generation; we only need the edge cases

### nemotron_swe: NEW → up to 10K
**Strategy:** Add SWE problem-solving diversity.
- Already in OpenAI format (minimal preprocessing)
- Real software engineering tasks with tool usage
- Complements swesmith (different problem distribution)

## Implementation Plan

1. **Update preprocessing scripts** to support quality filtering:
   - Add `--min-messages N` flag to filter by conversation length
   - Add `--min-assistant-tokens N` flag to filter by response length
   - Add `--max-rows N` flag (already exists in most scripts)
   - Add `--sort-by-length` flag to keep longest first

2. **Update `build_training_mix.py`** with new target counts

3. **Run preprocessing** on local machine (can use `uv run` for HF dataset downloads)

4. **Re-pretokenize** on Romulus (same process as before, ~33 minutes)

5. **Verify** the new mix covers all three priorities with spot-checks

| Metric | 262K (old) | ~140K (new) |
|--------|-----------|-------------|
| Packed examples | 42,976 | ~20-24K (est.) |
| Training time (1x H100) | ~176 hrs | ~80-95 hrs |
| Cost @ $2.15/hr | ~$378 | ~$172-204 |
| Avg quality per example | Medium (lots of simple examples) | Higher (filtered for complexity) |
| Risk of "washing out" | Higher (syntax noise dilutes learning) | Lower (every example teaches something new) |
