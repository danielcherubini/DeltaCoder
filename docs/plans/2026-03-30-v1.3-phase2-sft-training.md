## Why This Run Will Be Different From v1.2

| | v1.2 | v1.3 |
|--|------|------|
| seq_len | 8192 | **32768** |
| OCR sequences truncated | ~32,500 (50%) | **0** |
| Total steps (estimated) | ~23,056 | **~5,800** |
| Step time (slow, OCR) | ~8.8s | **~25-35s** |
| Step time (fast, short seqs) | ~0.5s | **~2s** |
| Peak VRAM (estimated) | 45GB | **~90-120GB** |
| Total training time | ~46hrs | **~45-55hrs** |
| Cost | ~$87 | **~$90-105** |

The step count drops by ~4x because 32768-token windows pack 4x more data. Step time increases proportionally. Net training time is similar.

**Critical:** Run the dry run (20 steps) BEFORE launching full training to verify VRAM doesn't OOM at 32768. If it OOMs, fall back to `sequence_len: 16384`.

| Parameter | Value |
|-----------|-------|
| **Framework** | Axolotl 0.16.0 |
| **Base model** | `Qwen/Qwen3.5-9B` |
| **Method** | LoRA (r=64, alpha=32) |
| **Sequence length** | **32768** |
| **Sample packing** | true |
| **Micro batch size** | 1 (GDN limitation — cannot go higher with packing) |
| **Gradient accumulation** | 4 |

| **group_by_length** | true |
| **Hardware** | NVIDIA H200 141GB (Vast.ai, France) |
| **Dataset** | 230K rows (same as v1.2, re-tokenized at 32768) |
| **Estimated steps** | ~5,800 |
| **Estimated training time** | ~45-55 hours |
| **Estimated cost** | ~$90-105 |

### Important Notes on group_by_length at 32768

With `group_by_length: true` and `sequence_len: 32768`:
- OCR sequences (avg 8,477 tokens) will be packed together — multiple per window
- Previously at 8192, each OCR sequence filled its own window
- At 32768, ~3-4 OCR sequences fit per window = dramatically fewer OCR-territory slow steps
- The "slow phase" will be much shorter than v1.2's ~57% of steps
- CoderForge trajectories (which can be 4K-16K tokens) will also pack better

**Eval sub-progress bars** (the 1884-step inner bars seen in v1.2): These show the eval dataset being scored. With `evals_per_epoch: 4` and `val_set_size: 0.02`, the eval set has ~4,600 rows. Packed at 32768, the eval bar will show fewer steps than v1.2's 1884 — this is expected.

### Step 2: Merge SFT LoRA into base model (on H200)
```bash
ssh -T -o StrictHostKeyChecking=no -p 28988 root@213.5.130.43

python3 -c "
from peft import PeftModel
from transformers import AutoModelForCausalLM, AutoTokenizer
import torch

print('Loading base model...')
base = AutoModelForCausalLM.from_pretrained(
    'Qwen/Qwen3.5-9B',
    torch_dtype=torch.bfloat16,
    device_map='cpu',
    trust_remote_code=True,
    attn_implementation='eager',
)
print('Loading SFT adapter...')
model = PeftModel.from_pretrained(base, '/workspace/DeltaCoder/outputs/deltacoder-9b-v1.3')
print('Merging...')
model = model.merge_and_unload()
print('Saving...')
model.save_pretrained('/workspace/merged_v1.3')
AutoTokenizer.from_pretrained('Qwen/Qwen3.5-9B', trust_remote_code=True).save_pretrained('/workspace/merged_v1.3')
print('Done')
"
```
