# DeltaCoder v1.2 — Phase 1: Dataset Preprocessing

**Status:** ✅ COMPLETED

**Goal:** Preprocess 5 coding-focused datasets into a standardized JSONL format and merge them into a single training file for SFT.

**Architecture:** One preprocessing script per dataset, each producing a JSONL file with a standard schema. Then a merge script combines and shuffles all files.

**Tech Stack:** Python 3, `datasets` library (HuggingFace), `json`, `re`

---

## Dataset Selection

### What changed from the original plan

The original plan included 7 datasets with ~215K rows. After analysis, we made these changes:

| Original Dataset | Action | Reason |
|-----------------|--------|--------|
| Glaive Function-Calling v2 (50K) | ❌ **Dropped** | Generic chatbot function calling (calculator, weather). Not coding-relevant. |
| Hermes Function-Calling v1 (11.6K) | ❌ **Dropped** | Same issue — mixed generic tool calling. |
| Opus Reasoning (2.3K) | ❌ **Dropped** | Too small to matter (1.1%). Superseded by OpenCodeReasoning. |
| Qwen3.5 Reasoning (633) | ❌ **Dropped** | Too small (0.3%). Superseded by OpenCodeReasoning. |
| nvidia/OpenCodeReasoning (65K) | ✅ **Added** | R1-generated competitive coding with `<think>` reasoning traces. Execution-verified. |
| Salesforce/xlam-function-calling-60k (15K) | ✅ **Added** | Verified function calling with code-relevant tools. |

### Final v1.2 dataset mix (230K rows, 100% coding-focused)

| Dataset | Rows | % | Purpose |
|---------|------|---|---------|
| nvidia/OpenCodeReasoning split_0 | 65,000 | 28.3% | Code reasoning with `<think>` traces |
| ise-uiuc/Magicoder-OSS-Instruct-75K | 50,000 | 21.7% | Single-turn code generation |
| togethercomputer/CoderForge-Preview | 50,000 | 21.7% | SWE-agent tool-use trajectories |
| m-a-p/Code-Feedback | 50,000 | 21.7% | Multi-turn code revision |
| Salesforce/xlam-function-calling-60k | 15,000 | 6.5% | Verified function calling |

## Output Schema (ALL preprocessed files use this)

Every preprocessed JSONL file has one JSON object per line:
```json
{"messages": [{"role": "user", "content": "..."}, {"role": "assistant", "content": "..."}], "source": "dataset_name", "id": "unique_id"}
```

For tool-calling examples:
```json
{"messages": [...], "tools": [{"type": "function", "function": {"name": "...", "description": "...", "parameters": {...}}}], "source": "dataset_name", "id": "unique_id"}
```

Valid roles: `system`, `user`, `assistant`, `tool`.

## Notes

- xlam dataset is gated on HuggingFace — requires accepting terms and setting `HF_TOKEN`
- OpenCodeReasoning's R1 reasoning traces are very long (many fill 8K-32K tokens). NVIDIA trained their models with 32K context. We use 8192.
- `pretokenize.py` normalizes xlam `tool_calls` fields into `<functioncall>` content strings for `apply_chat_template` compatibility
