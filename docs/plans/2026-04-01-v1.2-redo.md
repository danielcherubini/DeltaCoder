# DeltaCoder v1.2 — Redo Plan (REVISED — Post DDP Validation)

**Goal:** Redo v1.2 SFT + DPO at **32K context** with proper tool-calling data, targeting the BEST 9B for (1) Coding, (2) Tool Calling, (3) Agentic Workflows. Must preserve vision capabilities.

**Architecture:** Unsloth FastVisionModel (text-only LoRA with packing) -> SFT at 32K on multi-GPU -> validate -> DPO -> merge -> GGUF -> test on Kronk -> upload to HuggingFace.

**Tech Stack:** Unsloth (SFT), vLLM nightly (DPO pair gen), TRL (DPO training), ik_llama.cpp (GGUF export)

**Status:** DDP multi-GPU validated on 2x A100 SXM4. Revised dataset (~157K rows, ~700M tokens) with Jackrong-inspired quality filtering. Waiting for post-Easter GPU deals to launch full training.

---

## What went wrong in the first attempt

1. `normalize_messages()` in `pretokenize.py` corrupted ~65K+ tool-calling examples (converted structured `tool_calls` into raw text)
2. Trained at 8K context (truncated reasoning traces)
3. No validation gate before DPO — wasted money on degraded DPO pairs

## What changed in the redo

1. **32K context** instead of 8K — full reasoning traces preserved
2. **Pruned 149K dataset** — quality-filtered from 262K, avg 9,455 tokens/row (nearly 2x denser)
3. **Unsloth instead of Axolotl** — Axolotl OOMs at 32K (Liger fused CE doesn't patch VL model), Unsloth handles it internally
4. **FastVisionModel + finetune_vision_layers=False** — preserves vision weights while only training language layers
5. **VLM packing unblock** — Unsloth blocks packing for VLMs; we patch trainer.py to remove the `is_vlm` check (validated in unslothai/unsloth#4160)
6. **Pre-tokenized parquet dataset** — tokenized offline on Romulus, avoids on-GPU tokenization
7. **DDP multi-GPU support validated** — Unsloth DDP works with our FastVisionModel + packing setup
8. **Validation gates** — test SFT model before DPO, test on Kronk before uploading

## Revised Training Approach (Jackrong-Inspired)

**Updated 2026-04-05** based on analysis of Jackrong's Qwopus3.5 work (+4.87pp HumanEval on Qwen3.5-9B from focused, quality-filtered data).

### Philosophy: Quality > Quantity

Jackrong demonstrated that ~14K highly curated rows outperforms larger, noisier datasets.
Key insight: the base model already knows code — we're sharpening specific capabilities, not teaching from scratch.

Applied principles:
- **Tiered token limits**: ≤8K for coding/tool-calling (concise, complete examples), ≤16K for agentic/SWE (inherently longer)
- **Train on responses only**: mask user/system tokens, only compute loss on assistant responses — prevents instruction memorisation, the Jackrong default
- **LoRA alpha=64** (1:1 ratio with r=64): stronger LoRA influence vs old 0.5:1 ratio
- **1 epoch**: 157K rows is already 10x Jackrong's dataset — no benefit from repeat passes
- **Drop nemotron_swe entirely**: 100% of its 10K rows exceed 16K tokens (median 43K tokens), were being severely truncated during pretokenization — zero useful signal

### New Dataset Mix (~157K rows, ~700M tokens)

**Tier 1: ≤8K tokens (Coding + Tool Calling)**

|--------|------|---|-------|

## Infrastructure

### Bootstrap (Validated on H100, A100, A6000)

The provisioning script (`v1.2/scripts/provision.sh`) sets up a fresh instance in ~4 minutes:
1. Creates Python 3.12 venv at `/workspace/venv/`
2. Installs Unsloth 2026.3.18 + all dependencies
3. Clones and patches causal-conv1d for detected GPU SM arch
4. Installs flash-linear-attention
5. Downloads scripts from GitHub (train_unsloth.py, patch_vlm_packing.py)
6. Applies VLM packing unblock patch
7. Pre-downloads Qwen3.5-9B tokenizer/config

**Provisioning auto-detects GPU architecture** — works on:
- SM 8.0 = A100 (Ampere)
- SM 8.6 = A6000 (Ampere)
- SM 8.9 = L40S (Ada Lovelace)
- SM 9.0 = H100/H200 (Hopper)

**To create an instance:**
```bash
# Search (adjust gpu_name and num_gpus as needed)
vastai search offers 'gpu_name=H100_SXM num_gpus>=2 dph<5 reliability>0.90' --order 'dph' --raw

vastai create instance <OFFER_ID> \
  --image vastai/pytorch:2.10.0-cu128-cuda-12.9-mini-py312-2026-03-26 \
  --env '-e DATA_DIRECTORY="/workspace/"' \
  --disk 80 \
  --ssh --direct

# SSH in and bootstrap:
mkdir -p /workspace
curl -fsSL -o /workspace/provision.sh https://raw.githubusercontent.com/danielcherubini/DeltaCoder/main/v1.2/scripts/provision.sh
bash /workspace/provision.sh

# Upload data:
scp -P <PORT> v1.2/data/v1.2_pretokenized_pruned.parquet root@<IP>:/workspace/v1.2_pretokenized_pruned.parquet
```

Note: `/workspace` may not exist on some images — create it first. The `PROVISIONING_SCRIPT` env var does NOT auto-run on `vastai/pytorch` images.

### Training Commands

**Single GPU:**
```bash
source /workspace/venv/bin/activate
export HF_TOKEN="..." HF_HUB_ENABLE_HF_TRANSFER=0 HF_HOME="/workspace/.cache/huggingface"

python train_unsloth.py \
    --data /workspace/v1.2_pretokenized_pruned.parquet \
    --batch-size 1 --grad-accum 4 --lr 1e-4 \
    --output-dir /workspace/outputs/deltacoder-9b-v1.2
```

**Multi-GPU DDP:**
```bash
torchrun --nproc_per_node=<NUM_GPUS> train_unsloth.py \
    --data /workspace/v1.2_pretokenized_pruned.parquet \
    --batch-size 1 --grad-accum 4 --lr 1e-4 \
    --output-dir /workspace/outputs/deltacoder-9b-v1.2
```

**Important DDP notes:**
- `ddp_find_unused_parameters=False` is set in train_unsloth.py (required because frozen vision encoder creates unused params)
- Effective batch size = batch_size × grad_accum × num_GPUs
- With 4 GPUs and grad_accum=4, effective batch = 16. May want to reduce grad_accum to 2 for 4+ GPUs.
- Always use `nohup` for long-running training to survive SSH disconnects

### CUDA Matching (CRITICAL)
- `causal-conv1d` compiles CUDA kernels at install time
- Must use `vastai/pytorch:2.10.0-cu128-cuda-12.9-mini-py312-2026-03-26` (toolkit 12.9, torch 12.8)
- NGC `26.01-py3` has toolkit 13.1 but torch 12.8 → FAILS
- provision.sh auto-detects SM arch and patches causal-conv1d to only build for the current GPU

### Why Unsloth (not Axolotl)
| Aspect | Axolotl | Unsloth |
|--------|---------|---------|
| 32K OOM | YES — Liger fused CE only patches ForCausalLM | NO — handles fused CE internally |
| VLM support | Loads ForConditionalGeneration but loss OOMs | FastVisionModel preserves vision + fused CE |
| Packing | Works out of the box | Blocked for VLMs — needs patch |
| Multi-GPU | FSDP2 works (minus 32K OOM) | DDP via torchrun ✅ validated |
