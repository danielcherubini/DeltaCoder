# DeltaCoder v1.2 — Phase 3: DPO Training + GGUF Export

**Status:** ⏳ PENDING — waiting for Phase 2 (SFT) to complete

**Goal:** Generate on-policy DPO preference pairs from the v1.2 SFT model, run DPO training, merge both LoRA adapters, export all GGUF quants, and upload to HuggingFace.

**Prerequisites:**
- Phase 2 completed — v1.2 SFT LoRA adapter merged + uploaded to HF
- Same H200 instance from Phase 2 (TRL, PEFT already installed — just need vLLM)

**Architecture:** Merge SFT LoRA → serve via vLLM → generate pairs → DPO train (HF+PEFT+TRL) → two-stage merge → GGUF export.

**Tech Stack:** vLLM (inference), HuggingFace + PEFT + TRL (DPO training, NOT Unsloth), llama.cpp (GGUF export)

⚠️ **DO NOT use Unsloth DPOTrainer.** It has a known bug where it detects Qwen3.5 as a vision model and crashes with `KeyError: 'images'`. Use `scripts/train_dpo.py` which uses plain HuggingFace + PEFT + TRL.

### Task 1: Serve Merged SFT Model with vLLM for Pair Generation

**Context:**
The DPO pair generation script needs an OpenAI-compatible API endpoint serving the v1.2 SFT model. The merged model already exists at `/workspace/merged_v1.2` from Phase 2 post-training steps.

**On same H200 instance:**

**Step 1: Mount SFT model via hf-mount (skip re-downloading if already at /workspace/merged_v1.2)**
```bash
# Option A: Use local merged model (already on disk from Phase 2)
# /workspace/merged_v1.2 already exists — skip mount, go straight to vLLM

# Option B: Mount from HF instead (if local copy unavailable)
mkdir -p /mnt/sft-model
hf-mount start --hf-token $HF_TOKEN \
    repo danielcherubini/Qwen3.5-DeltaCoder-9B-v1.2-SFT /mnt/sft-model
# Then use /mnt/sft-model as --model path below
```

**Step 2: Serve with vLLM**
```bash
nohup python -m vllm.entrypoints.openai.api_server \
    --model /workspace/merged_v1.2 \
    --served-model-name deltacoder \
    --port 18000 \
    --host 0.0.0.0 \
    --dtype bfloat16 \
    --max-model-len 4096 \
    --enforce-eager \
    --enable-prefix-caching \
    > logs/vllm.log 2>&1 &

# Wait for server
sleep 60 && curl -s http://localhost:18000/health
```

**Step 3: Generate DPO pairs**
```bash
cd /workspace/DeltaCoder && nohup python scripts/generate_dpo_pairs.py \
    --n-problems 10000 \
    --n-samples 8 \
    --api-base "http://localhost:18000/v1" \
    --model deltacoder \
    --output data/dpo_pairs_v1.2.jsonl \
    --concurrency 32 \
    > logs/generate_dpo_v1.2.log 2>&1 &
```

Expected: ~4hrs, ~4500 pairs at 45% keep rate.

**Step 4: Download pairs locally**
```bash
rsync -avP -e "ssh -p <PORT>" \
    root@<IP>:/workspace/DeltaCoder/data/dpo_pairs_v1.2.jsonl \
    ~/Coding/AI/DeltaCoder/data/
```

**Acceptance criteria:**
- [ ] Merged model served via vLLM (health check passes)
- [ ] Pair generation completes 10K problems
- [ ] ~4000+ preference pairs in `data/dpo_pairs_v1.2.jsonl`
- [ ] Keep rate ~40-50%

### Task 2: DPO Training

**Context:**
Run DPO training on the v1.2 SFT model with the newly generated preference pairs. Uses `scripts/train_dpo.py` (plain HF+PEFT+TRL, no Unsloth).

**On same H200 instance:**

```bash

    --data data/dpo_pairs_v1.2.jsonl \
    --output-dir ./outputs/deltacoder-9b-v1.2-dpo \
    --max-steps 50
```

Verify:
- Loss starts near 0.693 and decreases
- rewards/margins are positive and growing
- rewards/accuracies > 0.5

```bash
    > logs/train_dpo_v1.2.log 2>&1 &
```
